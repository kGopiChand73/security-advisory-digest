# 🛡️ Security Advisory Digest

> **AI Prototype Challenge** — Category: *Infrastructure Maintenance*

An AI agent that turns noisy public security feeds into a focused
**only-what-affects-you** Markdown digest.

## 1. Problem statement

Vendor security feeds are **noisy** and relevance is **unclear**. Engineers
get hundreds of advisories each week; only a handful actually affect their
stack. This PoC pulls 3 public feeds, matches advisories against a
declared tech stack using **RAG**, and produces a short Markdown report
listing **only the items that matter** to the team.

## 3. Features implemented

- ✅ Pulls **3 public feeds**: GitHub Security Advisories, NVD (NIST), CISA KEV
- ✅ Loads tech inventory from **`stack.yaml`**
- ✅ **RAG** (TF-IDF cosine similarity) to find candidate advisories per stack item
- ✅ **LLM judge** (mock / Gemini / OpenAI) confirms true matches and emits **structured JSON**
- ✅ **Streamlit UI** — upload stack, run pipeline, preview & download digest
- ✅ Outputs **Markdown digest** + **structured JSON**
- ✅ Pytest / unittest happy-path tests with offline fixtures
- ✅ Zero-dependency core (stdlib only); Streamlit and LLM clients are optional

## 4. Architecture

```
            └────────────┬────────────────┘
                         │  ~90 advisories
                         ▼
   stack.yaml ──►  ┌────────────────┐
                   │ RAG vector store│  TF-IDF index
                   │  (in-memory)    │
                   └────────┬────────┘
                            │  top-K candidates
                            ▼
                   ┌────────────────┐
                   │  LLM judge     │  mock | Gemini | OpenAI
                   │ (structured JSON)│  → {affects, impact, action, confidence}
                   └────────┬────────┘
                            │  confirmed matches
                            ▼
                  outputs/digest_YYYY-MM-DD.md
                  outputs/digest.json (download)
```

## 7. Setup

```powershell
# 1. Clone & enter project
git clone https://github.com/<YOUR-ORG>/security-advisory-digest.git
cd security-advisory-digest

# 2. Create virtual env (Python 3.10+)
python -m venv .venv
.\.venv\Scripts\Activate.ps1     # Windows
# source .venv/bin/activate       # Linux/macOS

# 3. Install dependencies
pip install -r requirements.txt
```

> The CLI works **without installing anything** — Streamlit and Gemini
> are only needed for the optional UI / real-LLM features.

### Optional: enable Gemini (free tier)

1. Get a free key at https://aistudio.google.com/apikey
2. Install the client: `pip install google-generativeai`
3. Copy `.env.example` to `.env` and set:
   ```
   LLM_PROVIDER=gemini
   GEMINI_API_KEY=<your-key>
   GEMINI_MODEL=gemini-1.5-flash
   ```

## 10. AI capability demonstrated

1. **External API integration** — pulls live data from 3 public security APIs.
2. **RAG** — TF-IDF retrieval over fetched advisories per stack item.
3. **LLM with structured output** — judge returns strict JSON
   `{affects, impact, action, confidence}` for every candidate.
4. **Agent loop** — the app **decides** what to do at each step:
   *fetch → index → retrieve → judge → filter → write*.

## 11. Assumptions and limitations

- **Live feed availability** — the 3 public feeds must be reachable. The
  app degrades gracefully if one feed fails (warns + continues).
- **Mock judge accuracy** — the offline rule-based judge can produce
  false positives (matches on shared keywords). Switch to Gemini for
  better accuracy.
- **TF-IDF retrieval** — lexical, not embedding-based. Sufficient for the
  PoC; a future iteration could swap to sentence-transformers.
- **Stack must be declared** — anything missing from `stack.yaml` will
  not be checked.
- **English-only** advisory text is assumed.
- **Daily run** — designed for a single execution per day; no scheduling.
